Templates

Templates are inert chunks of DOM that can be reused throughout the page.

If you've used mustache or handlebars it's the same concept. But now it has its own tag.
Show of hands, how many of you have used mustache or handlebars?

Template Basics


<template id="my-widget">
  <p>Once upon a time...</p>
  <img src=""/> <!-- Fill me in when you're ready -->
  <script>
    alert("I'm chillin."); // Totally inert!
  </script>
</template>

The Content Keyword

JavaScript can not access the internals of our template unless we use the content keyword

Using Templates


<template id="my-widget">
  <img src=""/>
  <p>Templates. Full of WIN!</p>
  <script>
    alert("I'm alive!")
  </script>
</template>

<script>
  var template = document.querySelector('#my-widget');
  template.content.querySelector('img').src = './img/corgi.gif';
  document.body.appendChild(template.content.cloneNode(true));
</script>

Shadow DOM

The Shadow DOM provides style and markup encapsulation.

It is the same technology used by browser makers to implement tags like <video> and <textarea>.

<video>'s secret DOM

Important Terms

The <video> tag is the shadow host.

The node with all the content is the shadow root.

Nested Shadow DOM nodes are a shadow tree.

The content is protected by the shadow barrier.

Our first shadow root


<div class="widget">Hello, world!</div>
<script>
  var host = document.querySelector('.widget');
  var root = host.createShadowRoot();
  root.textContent = 'Im inside yr div!';
</script>
Hello, world!
Inspect with dev tools The takeaway is that the content inside of the shadow root is what gets rendered.

</script>
Jigglypuff
Using the <content> tag we’ve created an insertion point that projects the text from the .pokemon div so it appears within our shadow <h1>. Insertion points are very powerful because they allow us to change the order in which things render without physically altering the source. It also means that we can be selective about what gets rendered. You may have noticed that we’re using a template tag instead of building our Shadow DOM entirely in JavaScript. I’ve found that using <template> tags makes the process of working with the Shadow DOM much easier.
